--- layout: documentation permalink: /documentation/edit-academic-staff-page/ question: How to Edit an Academic Staff Profile page? index: 110 title: How to Edit an Academic Staff Profile page? ---
  1. Login to GitHub

  2. Go to the academic staff profile page that you want to edit on {% include documentation_image.html url_image='/images/docs/go_to_academic_staff_page.PNG' text='this' %} website

  3. At the bottom of the page, click {% include documentation_image.html url_image='/images/docs/edit_this_on_github_staff.PNG' text='"Edit this page on GitHub"' %}

  4. Click the {% include documentation_image.html url_image='/images/docs/click_edit_inside_github.png' text='"Edit"' %} icon

  5. Make your changes to the text in {% include documentation_image.html url_image='/images/docs/between_frontmatter.PNG' text='between' %} "---" according to the table below
    Variable Description
    layout Used internally. Do not change.
    permalink The link to the profile page. Do not change
    title The title of the webpage
    name_below_image Name of the person
    contact_number Contact number of the person
    email Email address of the person
    location Location (room) of the person, in the department building (Ex: 3-CE-01)
    text_below_name Text that will be shown below the name
    url_image Path to the image of the person in the Gihub Repo
    url_*site* URL of the profile pages in other *sites*
    designation Designation of the person. This is used to group the profiles in academic staff listing page. Only "Professor", "Senior Lecturer" and "Lecturer" are valid for now
    is_hod Used to determine if this is the Head of the Department. Only valid entries are "true" and "false".
    on_duty Is this person on duty ? Only valid entries are "true" and "false". If this is "false", the staff memeber will be considered as a past academic staff.
    research_interests Research interests of the person. Must be written as an array similar to a python list. Eg: ["interest1","interest2"]

  6. Make your changes to the text {% include documentation_image.html url_image='/images/docs/below_frontmatter.PNG' text='below' %} the second "---". This must be written in HTML and will be displayed to the {% include documentation_image.html url_image='/images/docs/right_of_profile_page.PNG' text='right' %} of the image in profile page. You can use Bootstrap 5 CSS Classes for styling .

  7. Click the {% include documentation_image.html url_image='/images/docs/propose_changes.png' text='"Propose changes"' %} at the bottom of the page

  8. Click the {% include documentation_image.html url_image='/images/docs/create_pull_req.png' text='"Create pull request"' %} button

  9. Click the {% include documentation_image.html url_image='/images/docs/create_pull_req2.png' text='"Create pull request"' %} button again

  10. Now an admin/a maintainer will approve these changes within 48hrs.
